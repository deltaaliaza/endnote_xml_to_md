## 🚀 快速部署

### GitHub Pages（推薦）

```bash
# 1. 上傳所有檔案到 GitHub Repository
# 2. 進入 Settings → Pages
# 3. 選擇 main 分支
# 4. 等待 1-2 分鐘

# 工具將在 https://yourusername.github.io/repository-name/ 可用
```

### 本地運行

```bash
# 方法 1：Python
python -m http.server 8000

# 方法 2：Node.js
npm install -g http-server
http-server

# 方法 3：直接打開
# 在瀏覽器中打開 index.html
```

## 📋 使用工作流

```
1. 在 EndNote 中
   ↓ File → Export → XML

2. 打開工具
   ↓ 上傳 XML 檔案

3. 檢查結果
   ↓ 查看統計和警告

4. 下載
   ↓ 點選「下載 ZIP」

5. 解壓
   ↓ 放到 Obsidian Vault

6. 在 Obsidian 中使用
   ↓ 編輯和連結
```

## 🚦 已知限制

### 設計限制
- 無法處理二進制附件
- 無離線 Service Worker（可擴展）
- 無深色模式（可擴展）
- 無拖放排序（可擴展）

### 瀏覽器限制
- IE 11 不支持
- 非常大的檔案（>1GB）可能導致記憶體問題
- 某些舊瀏覽器可能有兼容性問題

### 數據限制
- 非標準的 EndNote XML 格式可能無法解析
- 損壞的 XML 檔案會報錯
- 某些特殊字符可能需要手動修正
