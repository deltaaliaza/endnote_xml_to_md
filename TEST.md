# 測試和驗證指南

本文件包含完整的測試方法和邊界條件驗證說明。

## 測試環境準備

### 前置條件
- 現代瀏覽器（Chrome、Firefox、Safari、Edge）
- EndNote 或準備好的 XML 檔案
- 可選：Obsidian（用於檢查最終輸出）

### 瀏覽器相容性

| 瀏覽器 | 版本 | 支持 |
|--------|------|------|
| Chrome | 90+ | ✅ 完全支持 |
| Firefox | 88+ | ✅ 完全支持 |
| Safari | 14+ | ✅ 完全支持 |
| Edge | 90+ | ✅ 完全支持 |
| IE | 11 | ❌ 不支持 |

## 測試場景

#### 測試 1.1：檔案上傳
**目的**：驗證檔案上傳功能

步驟：
1. 打開工具
2. 點選上傳區域
3. 選擇有效的 XML 檔案
4. 確認顯示檔案名稱

預期結果：
- 檔案名稱在上傳區域下方顯示
- 無錯誤信息
- 工具開始解析

#### 測試 1.2：拖曳上傳
**目的**：驗證拖曳上傳功能

步驟：
1. 打開工具
2. 從檔案管理器拖曳 XML 檔案到上傳區域
3. 釋放滑鼠

預期結果：
- 上傳區域顯示 hover 狀態
- 檔案被上傳並解析

#### 測試 1.3：無效檔案
**目的**：驗證錯誤處理

步驟：
1. 嘗試上傳非 XML 檔案（如 .txt）
2. 嘗試上傳格式不正確的 XML

預期結果：
- 顯示適當的錯誤信息
- 不會導致頁面崩潰

#### 測試 2.2：XML 實體解碼
**目的**：驗證實體解碼功能

查看 `Al-Adwan-2021-Novel extension of the UTAUT mod.pdf` 記錄：
1. 檢查標題中的實體

#### 測試 2.3：Style 標籤移除

- 純文本輸出
- 無 XML 標籤或樣式信息

### 3. 檔名生成測試

#### 測試 3.1：從 PDF 生成檔名
**目的**：驗證 PDF 檔名優先級

查看所有有 PDF 的記錄：
```

- Al-Adwan-2021-Novel extension of the UTAUT mod.md
- Alfadda-2021-Measuring Students Use.md

```

檔案名稱應直接來自 PDF URL，去掉 `.pdf` 副檔名。

#### 測試 3.2：檔名清理
**目的**：驗證非法字符清理

檢查檔名中是否有：
- 正確的 UTF-8 字符（如中文）
- 移除的非法字符（`<>:"|?*`）
- 保留的合法字符（字母、數字、連字符、下劃線）

預期結果：
- 檔名在 Windows/Mac/Linux 上都合法
- 可讀性保持

#### 測試 3.3：檔名衝突處理
**目的**：驗證重複檔名處理

創建有意的重複檔名情景：
1. 使用編輯器修改 `app.js`，使兩個記錄生成相同檔名
2. 運行轉換
3. 檢查輸出

預期結果：
- 第一個為 `filename.md`
- 第二個為 `filename_2.md`
- 第三個為 `filename_3.md`
- 警告區域列出所有衝突

### 6. Markdown 輸出測試

#### 測試 6.1：檔案結構
**目的**：驗證 Markdown 檔案結構

檢查任何生成的 .md 檔案：
1. 確認以 `---` 開始
2. 確認 YAML 在兩個 `---` 之間
3. 確認所有必需的章節都存在

[content or empty]

## 邊界條件說明

本工具已處理以下邊界條件：

### 已處理
- ✅ XML 實體解碼（&amp;、&apos;、&#xD; 等）
- ✅ 超長檔名截斷
- ✅ 檔名衝突檢測和解決
- ✅ 多 PDF 附件處理
- ✅ 缺少 DOI 的記錄
- ✅ 缺少 PDF 的記錄
- ✅ 缺少作者的記錄
- ✅ 空字段處理
- ✅ UTF-8 和中文支持
- ✅ Windows/Mac/Linux 非法字符清理
- ✅ Style 標籤移除
- ✅ 特殊字符轉義在 YAML 中

### 未處理（設計限制）
- ❌ 損壞或非法格式的 XML（會顯示錯誤）
- ❌ 超級大的 XML（>1GB，瀏覽器記憶體限制）
- ❌ 舊版 EndNote 的非標準 XML 格式
- ❌ 二進制附件或圖片（不在 XML 範圍內）
